# 04 - Activity Route

## Parent
[PRD.md](../PRD.md)

## What to build
Implement the `/activity` route containing a searchable and filterable event log. The design should utilize `shadcn/ui` components (like Cards, Inputs, Tables/Lists) to present the event log instead of the old custom UI, ensuring a modern and clean aesthetic.

## Acceptance criteria
- [ ] `/activity` route is created and accessible.
- [ ] Event log is displayed using shadcn/ui components.
- [ ] Users can search/filter the event log.
- [ ] The page is fully responsive.
