# 02 - Auth, App Shell, & Routing

## Objective
Implement the routing structure and the responsive application shell.

## Tasks
1. Create a `_app.tsx` layout route for authenticated users.
2. Build the `Sidebar` component (left sidebar on desktop, bottom tab bar on mobile).
3. Build the `Topbar` component.
4. Create empty placeholder routes for `/dashboard`, `/activity`, `/devices`, `/alerts`, and `/settings`.
5. Create a simple `/signin` route and implement a mock auth state (e.g., using simple React Context or local storage) to protect the `_app` routes.
